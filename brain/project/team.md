## Contribución

Todos los miembros tienen acceso de push al repositorio. Antes de hacer cambios:

1. Crear rama con nombre descriptivo: `git checkout -b feature/nueva-feature`
2. Hacer commits pequeños y descriptivos
3. Enviar Pull Request al master para revisión
4. Mergear después de aprobación

Ver [workflow.md](../development/workflow.md) para más detalles.
